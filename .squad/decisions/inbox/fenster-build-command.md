### squad build uses SquadSDKConfig (not runtime SquadConfig)

**By:** Fenster (Core Dev)
**What:** The `squad build` command works with `SquadSDKConfig` from `builders/types.ts`, not the runtime `SquadConfig` from `runtime/config.ts`. Config files (`squad.config.ts`, `squad/index.ts`) should export a `SquadSDKConfig` for the build pipeline.
**Why:** `SquadSDKConfig` has the right shape for markdown generation — `TeamDefinition`, `AgentDefinition`, `RoutingDefinition`, `CeremonyDefinition`. The runtime `SquadConfig` is for the execution engine. Two separate concerns, two separate types.
**Impact:** Anyone writing a `squad.config.ts` for the build pipeline should use `defineSquad()` and `SquadSDKConfig`, not the runtime config type.

### Generated files stamped with HTML comment header

**By:** Fenster (Core Dev)
**What:** All files produced by `squad build` start with `<!-- generated by squad build — do not edit -->`. This makes it trivial to detect generated vs. hand-written files.
**Why:** Users need to know which files are safe to edit and which will be overwritten. The header also enables `--check` drift detection.
**Impact:** Parsers that consume team.md, routing.md, etc. should tolerate leading HTML comments.
